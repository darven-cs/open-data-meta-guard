# v2.0 需求文档

> 在 v1.0 元数据治理平台基础上的**精简重构版**。v1.0 是面向多阶段流水线（抓取 → 下载 → 评估 → 清洗 → 知识图谱 → 故事）的完整方案；
> v2.0 砍掉中间环节，聚焦「**采集 → 评估 → 上传 → 质量评估**」这条最短业务链，并补一个 Chat 入口做对话查询。
> 已落地的 v1.0 资产（core 基础设施、agent 编排机制、瑞士风前端）整体搬迁。

---

## 一、页面（5 个）

> v2.0 只保留 v1.0 中真正跑通的功能区。其它（知识图谱 / 数据故事 / 完整流水线编排）暂不纳入。

| # | 路由 | 中文名 | 英文名 | 功能描述 | 优先级 |
|---|---|---|---|---|---|
| 1 | `/meta-collect` | 元数据采集 | Metadata Ingestion | 输入 URL 列表 → 后端用 web_scrap agent 抓元数据 → 列表展示 + 查看详情 + 修改 + 删除 | P0 |
| 2 | `/meta-evaluate` | 元数据评估 | Metadata Evaluation | 选已采集的 dataset → 后端用 meta_evaluate agent 对元数据按 EU MQA 规则评分 → 生成 Markdown 评估报告 | P0 |
| 3 | `/data-collect` | 数据采集 | Data Acquisition | **人工上传**：下拉选已采集的 dataset → 选本地文件（csv/xlsx/json）→ 上传并落库 → 关联到对应 dataset | P0 |
| 4 | `/data-quality` | 数据质量评估 | Data Quality Evaluation | 选已上传的数据 → 后端用 data_quality_assess agent 对数据跑 pandera + ydata-profiling → 生成 Markdown 质量报告 | P0 |
| 5 | `/chat` | 开放数据小D | Data Agent | 对话式 LLM agent，基于已落库的元数据/数据回答用户开放数据相关问题 | P1（先做最简会话） |

> 移除 v1.0 的 HomeView Dashboard（5 个功能区不再是入口，但保留一个简易 HomeView 跳转列表）。

## 二、数据库（6 张表）

> **精简原则**：v2.0 只保留业务必需的实体。v1.0 的 13 张表里，`collect_tasks / clean_records / kg_build_records / data_stories / dataset_columns / tags / dataset_tags / tag_relations / quality_reports 独立表` 全部砍掉或并入评估表的 JSONB 字段。
> **唯一新增**：`chat_sessions` + `chat_messages`（v2.0 才有 Chat 入口，v1.0 有但 v2.0 文档未列）。

### 表清单

| # | 表名 | 业务实体 | v1.0 对应 |
|---|---|---|---|
| 1 | `datasets` | 元数据采集结果（数据集基本信息） | ✅ 同名表（v1.0 字段更多，加 `metadata` JSONB） |
| 2 | `data_downloads` | 上传的文件记录 | ✅ 同名表（v1.0 有，v2.0 字段精简 + 标记 `source='user_upload'`） |
| 3 | `meta_evaluations` | 元数据评估报告 | ✅ 同名表（v2.0 只存 `evaluation_content` md，规则明细塞 JSONB） |
| 4 | `data_quality_evaluations` | 数据质量评估报告 | 🆕 新表名（v1.0 是 `quality_reports`，改名 + 字段精简） |
| 5 | `chat_sessions` | 对话会话 | ✅ 同 v1.0 |
| 6 | `chat_messages` | 对话消息 | ✅ 同 v1.0 |

### 表结构

```sql
-- 1. datasets：元数据采集的核心实体
CREATE TABLE datasets (
    id           VARCHAR(64) PRIMARY KEY,                    -- sha256(url) 全 64 位 hex
    url          TEXT NOT NULL UNIQUE,                      -- 数据源 URL
    metadata     JSONB NOT NULL DEFAULT '{}'::jsonb,        -- web_scrap agent 抓回来的原始元数据（含 title/description/字段/规模等）
    status       VARCHAR(16) NOT NULL DEFAULT 'pending',    -- pending/scraped/evaluated/uploaded — 不加 CHECK
    created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_datasets_metadata ON datasets USING gin (metadata);
CREATE INDEX idx_datasets_status   ON datasets (status);

-- 2. data_downloads：人工上传的数据文件记录（v2.0 不再走 agent 下载）
CREATE TABLE data_downloads (
    id            BIGSERIAL PRIMARY KEY,
    dataset_id    VARCHAR(64) NOT NULL,                     -- 逻辑 FK -> datasets.id
    file_name     TEXT NOT NULL,                            -- 原始文件名
    file_path     TEXT NOT NULL,                            -- data/uploads/{id}.{ext} 相对路径
    file_format   VARCHAR(16) NOT NULL,                     -- csv/xlsx/json
    file_size     BIGINT,                                   -- bytes
    file_sha256   VARCHAR(64),                              -- 完整性校验
    source        VARCHAR(16) NOT NULL DEFAULT 'user_upload', -- v2.0 固定 user_upload,保留字段便于以后扩展 agent_download
    status        VARCHAR(16) NOT NULL DEFAULT 'uploaded',  -- uploaded/failed
    error_message TEXT,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_downloads_dataset ON data_downloads (dataset_id);
CREATE INDEX idx_downloads_status   ON data_downloads (status);

-- 3. meta_evaluations：元数据评估报告（基于 EU MQA）
CREATE TABLE meta_evaluations (
    id                  BIGSERIAL PRIMARY KEY,
    dataset_id          VARCHAR(64) NOT NULL,                -- 逻辑 FK -> datasets.id
    evaluation_content  TEXT NOT NULL,                       -- Markdown 报告全文
    rule_scores         JSONB NOT NULL DEFAULT '{}'::jsonb,  -- 每条 MQA 规则得分明细
    score_total         INTEGER,                             -- 0-405
    grade               VARCHAR(16),                         -- Excellent/Good/Sufficient/Bad
    created_at          TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_meta_eval_dataset ON meta_evaluations (dataset_id);
CREATE INDEX idx_meta_eval_grade   ON meta_evaluations (grade);

-- 4. data_quality_evaluations：数据质量评估报告
CREATE TABLE data_quality_evaluations (
    id                  BIGSERIAL PRIMARY KEY,
    dataset_id          VARCHAR(64) NOT NULL,                -- 逻辑 FK -> datasets.id
    data_download_id    BIGINT,                              -- 逻辑 FK -> data_downloads.id（被评估的文件）
    evaluation_content  TEXT NOT NULL,                       -- Markdown 报告全文
    summary             JSONB NOT NULL DEFAULT '{}'::jsonb,  -- 关键指标摘要：行数/列数/缺失率/重复率
    issues              JSONB NOT NULL DEFAULT '[]'::jsonb,  -- 高严重度问题列表
    created_at          TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_quality_eval_dataset  ON data_quality_evaluations (dataset_id);
CREATE INDEX idx_quality_eval_download ON data_quality_evaluations (data_download_id);

-- 5. chat_sessions：对话会话
CREATE TABLE chat_sessions (
    id          VARCHAR(64) PRIMARY KEY,                    -- uuid4 hex
    title       TEXT,                                       -- 首条消息摘要
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 6. chat_messages：对话消息
CREATE TABLE chat_messages (
    id          BIGSERIAL PRIMARY KEY,
    session_id  VARCHAR(64) NOT NULL,                       -- 逻辑 FK -> chat_sessions.id
    role        VARCHAR(16) NOT NULL,                       -- user/assistant/system
    content     TEXT NOT NULL,
    metadata    JSONB NOT NULL DEFAULT '{}'::jsonb,         -- SSE event 元数据：tool_call/reasoning/final
    created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_messages_session ON chat_messages (session_id);
```

### 与 v1.0 数据库的差异（迁移备忘）

| v1.0 表 | v2.0 决策 | 原因 |
|---|---|---|
| `datasets` | ✅ 保留，字段精简 | 核心实体 |
| `dataset_columns` | ❌ 删除，字段塞 `datasets.metadata` JSONB | v2.0 不做字段级独立查询 |
| `tags` / `dataset_tags` / `tag_relations` | ❌ 删除 | v2.0 不做标签体系 |
| `collect_tasks` | ❌ 删除 | v2.0 不做批次追踪（用户明确不需要） |
| `data_downloads` | ✅ 保留，新增 `source` 字段 | v2.0 只支持人工上传，但保留扩展位 |
| `clean_records` | ❌ 删除 | v2.0 不做数据清洗 |
| `quality_reports` | 🆕 改名 `data_quality_evaluations` | 与元数据评估表命名对齐 |
| `meta_evaluations` | ✅ 保留，字段精简 | 核心实体 |
| `data_stories` | ❌ 删除 | v2.0 不做数据故事（用户说先不做） |
| `kg_build_records` | ❌ 删除 | v2.0 不做知识图谱 |
| `chat_sessions` / `chat_messages` | ✅ 保留 | v2.0 入口之一 |

## 三、后端架构

### 3.1 技术栈

- **语言**：Python 3.13（与 v1.0 一致）
- **包管理**：uv（与 v1.0 一致）
- **Web 框架**：FastAPI + uvicorn（v1.0 已验证）
- **数据库**：PostgreSQL（v1.0 已配 docker-compose）
- **ORM**：SQLAlchemy 2.x async + asyncpg（v1.0 已配）
- **迁移**：alembic（async 模板，v1.0 已验证）
- **LLM**：langchain + ChatOpenAI 兼容协议（DeepSeek 等），`core/llm.py` 工厂模式
- **Agent 框架**：langgraph（v1.0 的 `nodes/collect_task_graph.py` 验证可行）
- **数据抓取**：Playwright（v1.0 已配 chromium + 多 context 池）
- **数据评估**：pandera（schema 校验）+ ydata-profiling（v1.0 已用）
- **日志**：loguru（v1.0 已配）
- **配置**：pydantic-settings + .env

### 3.3 Agent 清单（v2.0）

| Agent | 来源 | 用途 |
|---|---|---|
| `web_scrap` | 从 v1.0 搬迁 | 输入 URL → 输出 JSON 元数据 |
| `meta_evaluate` | 从 v1.0 搬迁 | 输入元数据 → 按 EU MQA 规则评分 → 输出 Markdown 报告 |
| `data_quality_assess` | 从 v1.0 搬迁（v1.0 叫 `quality_assess`） | 输入文件路径 + 数据 → pandera + ydata-profiling → 输出 Markdown 报告 |
| `chat` | v1.0 `orchestrator` 精简版 | 输入用户问题 + 会话历史 → 输出回答（SSE 流式） |

> **砍掉**：v1.0 的 `download` / `clean` / `kg_build` / `data_story` 全部不在 v2.0 范围。

### 3.4 API 设计（5 组）

#### 3.4.1 元数据采集 `/meta-collect`

| 方法 | 路径 | 功能 |
|---|---|---|
| POST | `/meta-collect/tasks` | 批量提交 URL（{urls: [...] }），返回 task_id 列表 |
| GET | `/meta-collect/datasets` | 列表（page/size，支持按 status/url 过滤） |
| GET | `/meta-collect/datasets/{id}` | 单条详情 |
| PUT | `/meta-collect/datasets/{id}` | 修改 metadata 字段 |
| DELETE | `/meta-collect/datasets/{id}` | 删除（级联删评估记录） |
| GET | `/meta-collect/tasks/{id}/status` | 异步任务进度查询（v2.0 可选，先做同步阻塞版） |

#### 3.4.2 元数据评估 `/meta-evaluate`

| 方法 | 路径 | 功能 |
|---|---|---|
| POST | `/meta-evaluate` | {dataset_id} → 触发评估，返回 evaluation_id |
| GET | `/meta-evaluate/{id}` | 单条评估详情 |
| GET | `/meta-evaluate?dataset_id=xxx` | 按 dataset 查历史评估列表 |

#### 3.4.3 数据采集（人工上传）`/data-collect`

| 方法 | 路径 | 功能 |
|---|---|---|
| GET | `/data-collect/datasets` | 列出**已采集元数据**的 datasets（供前端下拉选） |
| POST | `/data-collect/upload` | multipart: dataset_id + file → 落盘 + 算 sha256 + 写 data_downloads |
| GET | `/data-collect/{id}` | 单条上传记录详情 |
| GET | `/data-collect?dataset_id=xxx` | 按 dataset 查上传列表 |
| DELETE | `/data-collect/{id}` | 删除上传（同时删文件） |

#### 3.4.4 数据质量评估 `/data-quality`

| 方法 | 路径 | 功能 |
|---|---|---|
| POST | `/data-quality` | {data_download_id} → 触发评估，返回 evaluation_id |
| GET | `/data-quality/{id}` | 单条详情 |
| GET | `/data-quality?dataset_id=xxx` | 按 dataset 查历史评估列表 |

#### 3.4.5 对话 `/chat`

| 方法 | 路径 | 功能 |
|---|---|---|
| GET | `/chat/sessions` | 列出所有会话 |
| POST | `/chat/sessions` | 新建会话（可选初始 title） |
| DELETE | `/chat/sessions/{id}` | 删除会话 |
| GET | `/chat/sessions/{id}/messages` | 历史消息分页 |
| POST | `/chat/sessions/{id}/messages` | 发送消息，返回 **SSE 流**（v1.0 已验证 fetch-event-source） |
