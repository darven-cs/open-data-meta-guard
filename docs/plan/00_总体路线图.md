## 0. 项目背景与决策

### Context

v1.0 是面向多阶段流水线（抓取 → 下载 → 评估 → 清洗 → KG → 故事）的完整方案，包含 13 张表、8 个 agent、复杂批次编排。v2.0 砍掉中间环节，聚焦「采集 → 评估 → 上传 → 质量评估」最短业务链 + 简化 Chat 入口。

### 已确认的核心决策

| # | 决策点 | 选定方案 |
|---|--------|----------|
| 1 | 执行节奏 | **严格串行**，一 Phase 一 Phase 推 |
| 2 | 数据库迁移 | **按功能分多次迁移**（每个 Phase 一个 alembic 文件） |
| 3 | Docker 环境 | **v1.0 容器+数据卷全删**，重新启动 v2.0 独立容器 |
| 4 | 前端搭建方式 | **从零搭建**，只搬 v1.0 的可复用组件代码 |
| 5 | 元数据采集同步/异步 | **全部同步阻塞**，不走后台 |
| 6 | Chat agent 能力范围 | **本次只搭骨架预留接口**，能力后续扩展 |
| 7 | Chat 工具层 | **单一职责封装**，便于后续挂载查询工具 |

## 4. 执行规则

- 一 Phase 一 Phase 推。完成本 Phase 验收清单全部勾完后，再进入下一 Phase。
- 每个 Phase 独立 alembic 迁移文件，不合并。
