### 全链路 smoke test

```bash
# 1. 元数据采集
curl -X POST http://localhost:8001/meta-collect/datasets/ingest \
  -H "Content-Type: application/json" -d '{"urls": ["https://example.com"]}'

DATASET_ID=<上面返回的 dataset_id>

# 2. 元数据评估
curl -X POST http://localhost:8001/meta-evaluate \
  -H "Content-Type: application/json" -d "{\"dataset_id\": \"$DATASET_ID\"}"

# 3. 数据采集（上传）
curl -X POST http://localhost:8001/data-collect/upload \
  -F "dataset_id=$DATASET_ID" -F "file=@/path/to/test.csv"

DOWNLOAD_ID=<返回的 download id>

# 4. 数据质量评估
curl -X POST http://localhost:8001/data-quality \
  -H "Content-Type: application/json" -d "{\"data_download_id\": $DOWNLOAD_ID}"

# 5. Chat 骨架
curl -X POST http://localhost:8001/chat/sessions
curl -X POST http://localhost:8001/chat/sessions/$SESSION_ID/messages \
  -H "Content-Type: application/json" -d '{"content": "hello"}'
```

## 四、风险与回退

| 风险 | 缓解 |
|------|------|
| Playwright 冷启动慢导致 Phase 1 同步接口超时 | 复用 v1.0 已验证的多 context 池 + WAF 等待策略 |
| LLM 调用慢导致元数据采集单 URL 超 60s | 客户端合理超时；失败 URL 单独标记 status=failed |
| pandera + ydata-profiling 对大文件慢 | 限制 sample_size；失败时降级为基础统计 |
| Chat SSE 改造为非流式时前端需调整 | Phase 5 评估时按需保留 SSE 切换能力 |
| v1.0 数据卷全删后开发数据丢失 | 按 Phase 走，每 Phase 验证后立即进入下一 Phase |
